信号量和管程

同步

  • 并发问题：竞态条件
  • 同步
    • 多线程共享公共数据的协调执行
    • 包括互斥和条件同步
    • 互斥：在同一时间只有一个线程可以执行临界区
  • 确保同步正确很难
    • 需要高层次的编程抽象，比如锁
    • 从底层硬件支持编译，如禁用中断，原子操作等

信号量（semphere）

  • 信号量用一个整形（sem）表示，有两个原子操作
    • P（），sem减1，如果sem<0,等待，否则继续
    • V（）：sem加1，如果sem<=0,唤醒一个等待的P

信号量的使用

  • 信号量是整数

  • 信号量是被保护的变量

    • 初始化完成后。唯一改变一个信号量的值的办法是通过PV操作
    • 操作必须是原子

  • P操作会阻塞，V操作不会阻塞

  • 两种类型信号量

    • 二进制信号量：0或者1
    • 计数信号量：可取任何非负值

  • 用二进制信号量实现的互斥，初始值设为1

  • 用二进制信号量实现的调度约束：初始值设为0

信号量的实现

  • 使用硬件原语
    • 禁用中断
    • 原子指令
  • 用途
    • 互斥和条件同步
    • 等待条件是独立的互斥
  • 缺点
    • 程序员必须精通信号量
    • 容易出错
      • 使用的信号量已经被另一个线程占用
      • 忘记释放信号量
    • 不能处理死锁的问题

管程

  • 目的
    • 分离互斥和条件同步的关注
  • 管程
    • 一个锁：指定临界区
    • 0或者多个条件变量：等待/通知信号量用域管理并发访问共享数据
  • 一般方法
    • 收集在对象/模块中的相关共享数据
    • 定义方法来访问共享数据
  • 实现
    • lock
    • 条件变量
      • 需要维持每个条件队列
      • 线程等待的条件等待signal()

经典同步问题

  • 读者-写者问题
    • 读者：只读数据集
    • 写者：可以读取和写入
    • 共享数据：
      • 数据集
      • 信号量CountMutex初始化为1
      • 信号量WriteMutex初始化为1
      • 整数Rcount初始化为0
  • 哲学家问题
    • 一定的数据结构，来描述每个哲学家的当前状态
    • 该状态时一个临界资源，各个哲学家对他的访问是互斥的进行
    • 一个哲学家吃饱后，可能唤醒邻居，存在同步的关系
