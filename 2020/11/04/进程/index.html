进程

进程描述

  • 进程的定义：一个具有一定独立功能的程序在一个数据集合上的一次动态执行过程

  • 进程的组成：

    • 程序代码
    • 程序处理的数据
    • 程序计数器中的值，只是下一条将运行的指令
    • 一组通用的寄存器的当前值，堆，栈
    • 一组系统资源，比如打开的文件

  • 进程和程序的联系

    • 程序是产生进程的基础
    • 程序的每次运行构成不同的进程
    • 进程是程序功能的体现
    • 通过多次执行，一个程序可以对应多个进程，通过调用关系，一个进程可包括多个程序

  • 进程和程序的区别

    • 今晨是动态的，程序是静态的，程序是有序代码的集合；进程是程序的执行，进程有核心态/用户态
    • 进程是暂时的，程序是永久的，进程是一个状态变化的过程，程序可长久保存
    • 进程和程序的组成不同：进程的组成包括程序、数据和进程控制块

  • 进程的特点

    • 动态性：可动态的创建、结束进程
    • 并发性：进程可以被独立调度并占用处理机运行
    • 独立性（通过页表实现）：不同进程的工作相互之间不影响
    • 制约性：因访问共享数据/资源或进程间同步而产生制约

  • 描述进程的数据结构：进程控制块PCB

    • 进程控制块：操作系统管理控制进程运行所用的信息集合。操作系统用PCN来描述进程的基本情况以及运行变化的过程。PCB是进程存在的唯一标志。
    • PCB的组成：
      • 进程标识信息。本进程的标识，父进程的标识，用户标识
      • 处理及状态信息保存区：保存进程的运行现场信息
        • 用户可见寄存器
        • 控制和状态寄存器。程序计数器，程序状态字
        • 栈指针，过程调用、系统调用、中断处理和返回会用到该信息
      • 进程控制信息
        • 调度和状态信息，用域操作系统调度进程并占用处理机使用
        • 进程间通信信息
        • 存储管理信息
        • 进程所用的资源
        • 有关数据结构连接信息
    • PCB的组织方式
      • 链表：同一状态的进程其PCB成一个链表，多个状态对应多个不同的链表
      • 索引表：同一状态的进程归入一个index表，多个状态对应多个不同的index表

进程状态

  • 进程的生命周期
    • 创建
      • 系统初始化
      • 用户请求创建一个新进程
      • 正在运行的进程执行了创建进程的系统调用
    • 运行
      • 内核选择一个就绪的进程，让他占用处理机并执行，为何选择？怎么选择？
    • 等待：进程只能自己阻塞自己，因为只有进程自身才能直到何时需要等待某种事件的发生
      • 请求并等待系统服务，无法马上完成
      • 启动某种操作。无法马上完成
      • 需要的数据没有到达
    • 唤醒：进程只能被别的进程或者操作系统唤醒
      • 被阻塞的线程需要的资源得到满足
      • 被阻塞的进程等待事件到达
      • 将该进程的PCB插入到就绪队列
    • 结束
      • 正常退出（自愿）
      • 错误退出（自愿）
      • 致命错误（强制）
      • 被其他进程所杀（强制）
  • 进程的状态变化模型
    • 运行状态
    • 就绪状态
    • 等待状态
  • 进程挂起模型
    • 进程挂起状态，意味着进程没有占用内存空间，处在挂起状态的进程映像在磁盘上
    • 分类
      • 内存
        • 阻塞挂起
        • 就绪挂起
        • 运行到就绪挂起
      • 外存
        • 阻塞挂起到就绪挂起
    • 就绪挂起到就绪
    • 阻塞挂起到阻塞
    • 状态队列
      • 由操作系统维护一组队列，标识系统当中所有进程的当前状态
      • 不同的状态分别用不同的队列表示
      • 每个进程的PCB都根据它的状态加入到相应的队列中，当一个进程的状态发生变化时，它的PCB从一个状态队列中脱离出来，加入到另外一个队列

线程

  • 为什么使用线程
  • 什么是线程
    • 是进程中的一条执行流程
    • 资源角度
      • 进程把一组相关的资源组合起来，构成了一个资源平台
    • 运行的角度
      • 代码在这个资源平台上的一条执行流程
  • 线程控制块TCB
    • PC
    • SP
    • State
    • Register
  • 线程的优点
    • 一个进程中可以同时存在多个线程
    • 各个线程之间可与并发的执行
    • 各个线程之间可以共享地址空间和文件等资源
  • 线程的缺点
    • 一个线程崩溃，会导致其所属进程的所有线程崩溃
  • 线程和进程的比较
    • 进程是资源分配的单位，线程是CPU调度单位
    • 进程拥有一个完整的资源平台，线程只独享必不可少的资源，如寄存器和栈
    • 线程同样具有就绪、阻塞和执行三种状态，同样具有状态之间的转换
    • 线程能减少并发执行的时间和空间开销
      • 线程的创建时间比较短
      • 线程的终止时间比较短
      • 同一进程内线程切换时间比进程短
      • 由于统一进程的各线程间共享内存和文件资源，可直接进行不通过内核的通信
  • 线程的实现
    • 用户线程：在用户空间实现
      • 操作系统看不到的线程
      • 实现
        • 由一组用户级的线程库函数来完成线程的管理
      • 缺点
        • 如果一个线程发起系统调用而阻塞，则整个进程在等待
        • 当一个线程开始运行后，除非主动交出CPU的使用权，否则他所在的进程当中的其他线程将无法运行
        • 由于时间片分别配给进程，故与其他进程比，在多线程时，每个线程得到i的时间片较少，执行会较慢
    • 内核线程（TCB在内核中）：在内核中实现
      • 操作系统管理的线程
      • 在操作系统的内核中实现的一种线程机制，由操作系统的内核完成线程的管理
        • 在支持内核线程的操作系统中，由内核维护进程和贤臣过的山下文信息
        • 线程的管理通过系统调用的方式进行，由内核完成，系统开销较大
        • 在一个进程中，如果某个内核线程发起系统调用被阻塞，并不会影响内核线程的运行
        • 时间片分配给线程，多线程的进程获得更多的CPU时间
    • 轻量级进程：在内核中实现，支持用户线程
      • 内核支持用户线程。一个进程可有一个或多个轻量级线程，每个轻量级进程由一个单独的内核线程来支持。
  • 用户线程和内核线程的对应关系
    • 多对一
    • 一对一
    • 多对多

上下文切换

  • 停止当前运行进程，并且调度其他进程
    • 必须在切换之前存储许多部分的进程上下文
    • 必须能够恢复，所以进程不能显示它曾经被暂停过
    • 必须快速
  • 需要存储什么上下文
    • 寄存器，CPU状态
    • 因为比较耗时，所以尽可能避免
  • 过程
    • 操作系统为活跃进程准备了进程控制块PCB
    • 操作系统将进程控块放置在一个合适的队列中

等待和终止进程
  • wait()系统调用是被父进程用来等待子进程的结束
    • 一个子进程向父进程返回一个值，所以父进程必须接收这个值并处理
    • wait()系统调用胆热你这个要求
      • 使父进程睡眠等待子进程的结果
      • 当子进程调用exit()的时候，操作系统解锁父进程，并且通过exit()传递得到的返回值作为wait调用的一个结果，如果此时没有子进程存活，wait()立刻返回
      • 如果父进程是僵尸等待，wait()立即返回其中一个值，并且解除僵尸状态
  • 进程结束执行后，调用exit()这个系统调用
    • 将这个程序的”如果“作为一个参数
    • 关闭打开的文件，连接等
    • 释放内存
    • 释放大部分支持进程的操作系统结构
    • 检查父进程是否存活
      • 如果是，则保留结果直到父进程需要，此时，进程没有真正死亡，只是进入了僵尸状态
      • 如果否，释放所有的数据结构，这个进程死亡
    • 清理所有等待的僵尸进程
  • 进程终止时最终的垃圾收集
