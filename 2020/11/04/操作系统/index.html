操作系统

操作系统的特点

  • 并发
  • 异步
  • 虚拟
  • 共享

通过中断通知操作系统切换进程

启动

  • CPU,内存，I/O(disk等)
  • 每部分作用
    • disk：存放OS
    • BIOS:基本的I/O处理，加电以后从一个固定的地址开始执行CS:IP=0xf000:fff0(CS:段寄存器，IP：指令寄存器)，通过该地址进行自检（POST，加电自检），寻找显卡和执行BIOS，之后将Bootloader放到内存中
    • Bootloader：加载OS，放在硬盘的第一个扇区（512字节）。通过BIOS将Bootloader放到内存中，x86系统是从0x7c00开始，往上512个字节。此时CPU执行权由Bootloader掌握，之后由Bootloader将操作系统从磁盘块读到内存中

中断，异常和系统调用

  • 各个部分的定义
    • 系统调用：应用程序主动向操作系统发出服务请求
    • 异常：非法指令或者其他换的处理状态，由应用程序产生
    • 中断：来自不同硬件设备的计时器和网络的中断，由外设产生
  • 为什么操作系统不能直接操作外设？
    • 计算机运行中，内核是被信任的第三方，安全的角度
    • 只有内核可以执行特权指令
    • 提供统一的接口，为了方便应用程序
  • 操作系统怎么设计和实现中断、异常和系统调用的，他们的区别和特点是什么
    • 源头：
      • 中断：外设
      • 异常：应用程序意想不到的行为
      • 系统调用：应用程序请求操作提供服务
    • 处理时间
      • 中断：异步的
      • 异常：同步
      • 系统调用：异步或者同步（异步体现在返回数据的时候，不在等待返回）
    • 响应
      • 中断：持续的，对用户应用程序时透明的
      • 异常：杀死或者重新执行意想不到的应用程序指令
      • 系统调用：等待和持续

  • 中断和异常处理机制

    • 中断
      • 硬件
        • 将内部、外部事件设置中断标记
        • 中断事件的ID
      • 软件
        • 保存当前的处理状态(即保存程序当前的状态，包括寄存器等)
        • 中断服务程序处理
        • 清除中断标记
        • 恢复之前保存的处理状态
    • 异常：产生异常的编号
      • 保存现场
      • 异常处理
        • 杀死产生异常的程序
        • 重新执行产生异常的指令
      • 恢复异常
    • 系统调用
      • 底层是win32 API或者post API
      • 用户调用库函数会触发系统调用，此时会涉及到用户态到内核态（可以完全控制操作系统）的转换
      • 产生系统调用的时候，应用程序和操作系统拥有各自的堆栈，所以状态切换还涉及到堆栈的切换

  • 跨越操作系统边界的开销

    • 在执行时间上的开销超过程序调用
    • 开销：
      • 建立中断/异常/系统调用号与对应服务例程映射关系的初始化开销（即需要建中断表）
      • 建立内核堆栈
      • 验证参数（安全考虑）
      • 内核态到用户态的地址空间，更新页面映射权限
      • 和内核态独立地址空间TLB

  • 面向I/O是通过中断来处理的

  • 面向应用程序时通过系统调用和异常来处理的

地址空间&地址生成

  • 地址空间的定义
    • 物理地址空间：硬件支持的地址空间
    • 逻辑地址空间：运行的程序所拥有的内存范围
  • 地址空间的生成
    • 逻辑地址空间的生成（编译器）
      • 编译
      • 汇编：将文件通过汇编转换为.o文件，这些文件起始地址都是0，此时完成逻辑地址的生成
      • 链接：一个程序由多个小的程序组成，通过linker将多个.o文件转换为.exe文件，此时，程序可以在内存中运行
      • 载入：通过loader将.exe文件载入内存，此时内存由一定的偏移量，可以直接访问相应的内存数据等
    • 物理地址生成（操作系统）
      • CPU方面
        • 取出逻辑地址的内存内容
        • 内存管理单元寻找逻辑地址和物理地址之间的映射
        • 控制器从总线发送在物理地址的内存内容的请求
      • 内存方面
        • 内存发送物理地址内内存的内容给CPU
      • 操作系统方面
        • 建立逻辑地址和物理地址之间的映射
  • 地址安全检查

  • 内存碎片问题

  • 分区的动态分配

    • 最佳适配

      • 最小化外部碎片
      • 优势

        • 比较简单
      • 劣势

        • 容易产生很多没有的微小碎片
    • 最差适配
      • 避免有太多微小的碎片
      • 优势：
        • 分配是中等尺寸效果最好
      • 劣势

非连续内存分配

连续内存分配的缺点

  • 分配给一个程序的物理内存是连续的
  • 内存利用率较低
  • 有外碎片、内碎片的问题

非连续分配的优点

  • 一个程序的物理地址空间是非连续的
  • 更好内存利用和管理
  • 允许共享代码和数据
  • 支持动态加载和动态链接

非连续分配的缺点

  • 如何建立虚拟地址和物理地址的转换
    • 软件方案
    • 硬件方案
      • 分段
      • 分页

非连续分配的方案

  • 分段：更好的分离和共享

    • 连续的逻辑地址空间和分段的物理地址空间
    • 程序的分段地址空间
    • 分段寻址方案
      • 段访问机制
        • 一个段是一个内存”块“
        • 程序访问内存需要：段号和段内偏移
      • 硬件实现方案
        • 逻辑地址：由段号和偏移组成
        • 段表（由操作系统建立）：由base和limit组成，其中段的尺寸是可变的

  • 分页

    • 分页地址空间
      • 划分物理内存到固定大小的帧，一般是的2的幂，一个物理内存物理地址是一个二元组：由帧号f（F位，一共有2^F个帧）和o帧内偏移（S位，每帧有2^S字节）组成，物理地址=2^S*f+o
      • 划分逻辑地址空间至相同大小的页
      • 转换逻辑地址为物理地址
        • 页表
        • MMU/TLB
    • 页寻址方案
      • 逻辑地址：页号，页内偏移
      • 页表（操作系统建立）：帧号
    • 页寻址机制
      • 页映射到帧
      • 页是连续的虚拟内存
      • 帧是非连续的物理内存
      • 不是所有的页都有对应的帧
    • 分页机制的性能问题
      • 访问一个内存单元需要2次内存访问
        • 一次用于获取页表项
        • 一次用于访问数据
      • 页表可能非常大
      • 可以通过缓存或者间接访问来解决

  • 页表

    • 每个运行的程序都有一个页表

      • 属于程序运行状态，会动态的变化
      • PTBR：页表基址寄存器
      • 一些特殊的标志位来标志地址是否合法（比如，是否有相应的物理地址与之对应）

分页机制性能问题解决

通过TLB缓存近期访问的页帧转换表项（解决时间问题）
  • TLB具备快速访问性能
  • TLB命中，物理页号可以很快被获取
  • 如果未命中，对应的表项被更新到TLB中（编程中尽可能将使用的资源集中在一起）
多级页表（解决空间问题）

页表

  • 快表
  • 二级/多级页表
  • 反向页表
    • 不是让页表和逻辑地址空间大小对应，而让页表与物理地址空间大小对应（由于逻辑地址空间增长快于物理地址空间增长）
    • 实现方案
      • 页寄存器：
        • 以页帧号为索引，页表项内容是页号（和页表相反）

虚拟内存

起因

  • 程序规模的增长的速度远大于存储器容量增长的速度
  • 理想中的存储器
    • 更大，更便宜，更快，非易失性存储器

覆盖技术

  • 如果程序太大，导致内存放不下，可以使用手动覆盖，只把需要的指令和数据保存在内存中
  • 目标：在较小的内存中运行较大的程序，常用于多道程序系统，与分区存储管理配合使用
  • 原理
    • 把程序按照其自身的逻辑结构，划分为若干个功能上相对独立的程序模块，那些不会同时执行的模块共享同一块内存区域，按时间先后运行。
    • 必要部分的代码和数据常驻内存
    • 不常用的平时放在外存中，需要时才放入内存中
    • 不存在调用关系的模块不必同时装入内存中，从而可以 相互覆盖，也就是这些模块共用一个分区
  • 缺点
    • 将程序划分为若干模块，费时费力
    • 覆盖木块从外存装入内存，实际上是用时间换取空间

交换技术

  • 如果程序太多，导致内存放不下，可以使用自动交换，把暂时不用的程序换到外存中
  • 方法
    • 将暂时不能运行的程序送到外存，从而获得空闲的内存空间
    • 操作系统把一个进程的整个地址空间的内容保存到外存中，而将外村中的某个进程的地址空间读入带内存中，换入换出内容的大小为整个程序的地址空间
  • 问题
    • 交换时机：只当内存空间不够或有不够的危险时换出
    • 交换区的大小：必须足够大
    • 程序换入时的重定位：最好采用动态地址映射的方法

虚存技术

  • 想在有限的内存中，以更小的页粒度为单位装入更多更大的程序，可以使用自动虚拟存储技术
  • 覆盖技术增加了程序员的负担，交换技术增加了处理器的开销
  • 程序的局部性原理
    • 程序在执行中的一个较短时期，执行的指令地址和指令操作数地址，分别局限于一定的区域
      • 时间局部性：一条指令的一次执行和下次执行，一个数据的当前访问和下次访问都集中在一个较短的时期内
      • 空间局部性：当前指令和邻近的几条指令，当前数据和邻近的几数据都集中在一个较小的区域内
  • 可以在页式或段式内存管理的记住上实现
  • 虚存技术的特征：
    • 大的用户空间：通过把为u里内存和外存结合，提供给用户的虚拟内存空间通常大于实际的物理内存
    • 部分交换：与交换技术相比，虚拟存储的调入和调出式对部分虚拟地址空间进行的
    • 不连续性：物理内存分配是不连续的，虚拟地址空间使用不连续
  • 实现
    • 虚拟页式内存管理
      • 页表完成逻辑也到物理页帧的映射
      • 大部分虚拟存储系统都采用虚拟页式存储管理技术，即在页式存储管理的基础上，增加请求调页和页面置换功能
      • 基本思路
        • 请求调页：当一个用户程序调入内存运行时，只装入部分页面，就可启动程序运行
        • 置换：运行过程中，发现要运行的程序或者要访问的数据不在内存中，则向吸引同发出却页中断请求，系统在处理这个中断时，将外村中相应的页面调入内存中，使程序能继续运行
        • 页表表项：
          • 驻留位：表示该页在内存还是外存
          • 保护位：表示该页做何种类型的访问，如只读，可读写等
          • 修改位：表示此页在内存中是否被修改过，当回收页面时，根据此位来决定是否写回外存
          • 访问位：如果该页面被访问过，则设置该位，用于页面置换算法
      • 缺页中断的处理过程：
        • 如果在内存中有空闲的物理页面，则分配以物理页帧f，转第四步，否则转第二步
        • 采用某种置换算法，选择一个将被替换的物理页帧f，他所对应的逻辑页为q，如果该页在内存期间被修改过，则需要写回外存
        • 对q所对应的页表项进行修改，驻留位置为0
        • 将需要访问你的页p装入物理页面f中
        • 修改p锁对应的页表项的内兄，把驻留位置为1，物理页帧号置为f
        • 重新运行被中断的指令

页面置换算法

功能

  • 发生缺页中断，需要调入新的页面而内存已满时，选择内存中那个物理页面被置换

目标

  • 尽可能减少页面的换入换出次数
  • 页面锁定：在页表中添加锁定标志位

局部页面置换算法

  • 最优页面置换算法
    • 思路:发生缺页中断的时候，对于保存在内存中的每一个逻辑页面，计算在他的下一次访问之前还需要等待多长时间，选择等待时间最长的作为被置换的页面
    • 实际中无法实现，主要作为其他算法的性能评价依据
  • 先进先出（FIFO）
    • 思路：选择在内存中驻留时间最长的页面
    • 性能较差，调出的页面有可能时经常要访问的页面
  • 最近最少使用（LRU）
    • 思路：发生缺页中断时，选择最久未使用的那个页面
    • 是最优置换算法的一个近似，依据是程序的局部性原理
    • 实现方法：
      • 系统维护一个页面链表，头插法，最近最久未使用的作为尾节点
      • 设置活动页面堆栈
  • 时钟页面置换算法
    • Clock页面置换算法，LRU的近似
    • 思路
      • 需要使用页表项中的访问位，当页面被装入内存的时候，该位初始化为0，如果该页面被访问，则置为1
      • 把各个页面组织成环形链表，指针指向最老的页面
      • 发生缺页中断的时候，考察指针指向的最老的页面，若访问位为0，立即淘汰；若访问位为1，则将其置为0，指针往下移动一格，如此下去，直到找到被淘汰的页面，然后将指针向下移动一格
  • 二次机会法
    • 修改Clock算法，使它允许脏页总是在一次时钟头扫描中保留下来
  • 最不常用算法（LFU）
    • 当发生缺页中断的时候，选择访问次数最少的那个页面，并淘汰
    • 实现方法：对每个页面设置一个访问计数器，没当一个页面被访问时，该页面的访问计数器加1.在发生缺页中断的时候。淘汰计数值最小的那个页面
Belady现象
  • 在采用FIFO算法的时候，有时候会出现分配的物理页面增加，缺页率反而提高的异常现象
  • 原因
    • FIFO算法的置换特征与进程访问内存的动态特征是矛盾的，与置换算法的目标是不一致的，因此，被它置换出去的页面并不一定是进程不会访问的

全局页面置换算法

  • 局部页面置换算法基于局部性原理的前提

  • 工作集：一个进程当前正在使用的逻辑页面的集合

  • 常驻集：指当前时刻，进程实际驻留内存中的页面集合

  • 区别

    • 工作集是进程运行中固有的性质，而常驻集取决于系统分配给进程的物理页面数目，依据所采用的页面置换算法
    • 如果一个进程的整个工作集都在内存中，进程就会顺利的运行，而不会造成太多的缺页中断
    • 进程常驻集的大小达到某个数目以后，再给他分配更多的物理页面，缺页中断率也不会明显下降

  • 工作集切换算法

  • 缺页率页面置换算法

    • 思路：常驻集大小可变，缺页率高的时候，增加工作集；反之。则减少工作集。
      • 采用全局页面置换的方式，发生缺页中断的时候，被置换的页面可以是在其他进程中，各个并发进程竞争地使用物理页面
      • 性能好，但增加了系统开销
    • 影响缺页率的因素：
      • 页面置换算法
      • 分配给进程的物理页面数目
      • 页面本身的大小
      • 程序的编写方法

  • 抖动问题

    • 如果分配给一个进程的物理页面太少，不能包含整个工作集，那么进程会造成很多的缺页中断，需要频繁的在内存和外存之间替换页面，从而使进程的运行速度变得很慢，称这种状态为”抖动“
    • 产生抖动的原因
      • 随着驻留内存的进程数目增加，分配给每个进程的物理页面数不断减少，缺页率上升，所以OS要选择一个适当的进程数目和进程需要的帧数，以便在并发水平和缺页率之间达到一个平衡
