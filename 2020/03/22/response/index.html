response

响应消息格式

数据格式

  • 响应行
    • 组成 :协议，响应状态码，状态码描述
    • 响应状态码：服务器告诉浏览器本次请求和响应的一个状态
      • 状态码分为5类，分别以1-5开头
      • 1xx : 服务器接收客户端消息但是没有接收完成，发送1xx码
      • 2xx：成功
      • 3xx: 重定向,302(重定向),304(访问缓存)
      • 4xx：客户端错误，404(请求路径没有 相应的资源)
      • 5xx:服务器端错误
  • 响应头
  • 响应空行
  • 响应体

response对象

  • 功能：设置响应消息

    • 设置响应行：包括格式个状态码
    • 设置响应头：setHeader(String name, String value)
    • 设置响应体：
      • 获取输出流
      • 使用输出流

  • 重定向的特点：

    • 地址栏发生变化

    • 重定向可以访问其他服务器的资源

    • 重定向是两次请求，不能使用request对象共享数据

  • 服务器输出数据到浏览器

    • 获取字符输出流：response.getWriter()
    • 输出数据
    • 乱码问题：使用response.setContentType()

  • 服务器输出字节数据到浏览器

    • 获取字节输出流

      response.getOutputStream()

    • 输出数据

      write("String".getBytes())

  • 验证码

ServletContext 对象

  • 概念：代表整个web应用和服务器来通信
  • 获取
    • 通过request获取： requset.getServletContext()
    • 通过HttpServlet获取：this.getServletContext()
  • 功能
    • 获取MIME类型
      • String getMimeType(filename)可以获取
    • 共享数据
      • setArrtibute(String name ,Object value)
      • getAttribute(String name )
      • removeAttribute(String name )
    • 获取文件的服务器路径
      • getRealPath(String path)
