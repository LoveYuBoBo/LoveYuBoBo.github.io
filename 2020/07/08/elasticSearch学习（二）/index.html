elasticSearch学习（二）

发表于 2020-07-08 | 分类于 elsticSearch | |
字数统计字: 1k | 阅读时长分钟 ≈ 3

mapping定义

注意事项

  • 只有使用post命令写入文档，es才会自动为其创建id，而使用put的时候不会创建
  • update一个文档，只能使用post，put只能用来index或者create
  • index一个已经存在的文档，旧的文档会被删除，新的文档再被写入且版本号加一
  • 当创建一个新的文档到不存在的索引中，默认情况会创建相应的索引，且可以设置mapping
  • ES7中合法的type是_doc
  • 精确值和全文本的本质区别在于是否会被Analyzer分词
  • Analyzer由三部分组成，分别是Character filter+Tokenizer+Token Filter
  • match和match_phrase的区别，``match中terms之间是or的关系，而match_phrase的terms是and的关系且term`之间的关系也会影响查询
  • 如果希望match_phrase匹配更多的结果，可以增加slot参数
  • 如果mapping的dynamic设置为stric，索引一个包含新新增字段的文档时会报错
  • 如果mapping的dynamic设置为false，索引一个包含新新增字段的文档时，文档会被索引，且新字段在_source中可见，但是该字段无法被搜索。
  • 不能所以修改字段类型，修稿字段类型都会造成reindex
  • 可以在mapping文件中为indexing和searching指定不同的Analyzer
  • 字段类型为text的时候，也不一定会被全文搜索，因为可以为其字段指定 not index

查询分类

轻量搜索

  • 直接使用get方式，传递文档id

  • 使用get+_search查询所有结果，也可以使用拼接的方式具体查询，例如get +_search?q=age:18

  • 使用查询表达式查询

  • 使用过滤器(年龄大于18的)

高级搜索

全文搜索

  • 使用query，结果会以得分排序(得到喜欢读书和写代码的记录)

短语搜索

  • 精确匹配使用match_phrase,短语之间是and的关系（查询既喜欢读书有喜欢写代码的人）

高亮搜索

  • 增加hightlights参数即可

数据分析

聚合

  • 按兴趣爱好聚合分类,查询结果统计出各个爱好的人数

  • 查询特定员工的爱好，使用组合查询

  • 查询特定爱好员工的平均年龄

集群原理

集群

  • 集群是一群cluster.name相同的结点组成，选举称为主节点的接待你负责索引的增删改和其他节点的增删，不参与文档级别的操作
  • 任何节点都可以处理或者转发用户的请求

集群健康

  • 通过get/_cluster/health获取集群的健康状况，其中status字段会显示健康状况，包括green yellow red,yellow表示主分片工作正常，而有副分片不正常，red表示有主分片不正常
  • 故障转移，使用多个结点，当某个结点出现问题，其他结点包含所有数据

扩容

  • 创建索引的时候确定了主分片的数量，这个数字也确定了索引能够存储的最大数据量
  • 副分片的数量是可以动态调整的，因为读操作可以被主分片和副分片处理，所以增加副分片的数量可以提高读的性能

故障处理

  • 如果某个主节点发生故障，集群首先会选举出一个主节点
