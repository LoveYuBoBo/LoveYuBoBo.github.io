elasticSearch学习（四）

发表于 2020-07-09 | 分类于 elasticSearch | |
字数统计字: 613 | 阅读时长分钟 ≈ 2

搜索

空查询及参数

  • 使用不带参数_search，返回结果中包含hits,took,shards,timeout等信息
  • 空查询会查询出集群中所有的文档

多索引，多类型

  • 指定索引查询
  • 当在单一的索引下进行搜索的时候，Elasticsearch 转发请求到索引的每个分片中，可以是主分片也可以是副本分片，然后从每个分片中收集结果。多索引搜索恰好也是用相同的方式工作的，只是会涉及到更多的分片

分页

  • 使用from和size指定查询分页，个请求经常跨越多个分片，每个分片都产生自己的排序结果，这些结果需要进行集中排序以保证整体顺序是正确的
  • 分布式系统中深度分页，在分布式系统中，对结果排序的成本随分页的深度成指数上升。这就是 web 搜索引擎对任何查询都不要返回超过 1000 个结果的原因。

映射和分析

  • 不同类型的字段的索引方式不完全相同
  • 倒排索引由文档中所有不重复的词的列表构成，对于每个词，由一个包含它的文档列表
分析器
  • 分析包括两个步骤
    • 将一块文本分成适合于倒排索引的独立的 词条 ，
    • 将这些词条统一化为标准格式以提高它们的“可搜索性”，或者 recall
  • 分析器可以完成上面的工作，实际执行中包括下面三个功能
    • 字符过滤器
    • 分词器
    • Token过滤器
  • 内置分析器
    • 标准分析器
    • 简单分析器
    • 空格分析器
    • 语言分析器
  • 当你查询一个 全文域时， 会对查询字符串应用相同的分析器，以产生正确的搜索词条列表。
  • 当你查询一个 精确值 域时，不会分析查询字符串，而是搜索你指定的精确值

数据类型

  • string,integer,boolean,short,byte,long,float,double,date

  • 对于非string的域，只需要指定其type即可，对于string域，默认是包含全文的，也就是说会被分词

  • 对于string最重要的属性是index和analyzer

    • index控制怎么索引这个字符串：analyzed以全文索引这个域，not_analyzed是精确索引，不会分词，no表示不索引这个域
    • analyzer指定分析器
