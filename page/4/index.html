redis

  • 概念：是非关系型数据库 NOSQL(redis,mangoDB,hbase…>)
  • 用途
    • 用作缓存
  • 数据结构
    • 存储的是键值对，key是字符串,value有五种结构 String map linkedlist set sorted
    • 增删改查
  • 持久化
    • RDB
      • 默认的方式，不需要配置，在一定时间内检测key的变化，然后持久化数据
    • AOF
      • 配置redis.windows.conf，可以每次操作以后进行持久化,appendonly 设置为yes
  • Java客户端：Jedis
  • Jedis连接池：JedisPool

JavaWeb三大组件

JavaWeb三大组件

Servlet

  • 概念：运行在服务器端的小程序，是一个接口，它定义了Java类被浏览器访问到的规则

  • 快速入门

    • 创建JavaEE项目
    • 定义一个类，实现Servlet接口
    • 实现抽象方法
    • 配置Servlet

  • 运行原理

    • 服务器接收到浏览器端的请求后，解析请求的URL路径，获取访问的servlet的资源路径
    • 查找web.xml文件，是否有对应的标签体内容
    • 如果有，查找全类名
    • tomcat会将字节码文件加载进内存，并创建对象
    • 调用器方法

    servlet中的生命周期

    • init方法
      • 被创建，执行init方法，只执行一次
        • 默认情况下，第一次被访问时被创建
        • 在web.xml中使用标签配置创建时机
          • 在第一次启动时创建，配置使用负数
          • 配置为0或者正数的时候，当服务器启动的时候被创建
      • init方法只执行一次，说明一个servlet在内存中只存在一个对象，servlet时单例的。
        • 所以当多个用户同时访问的时候，可能存在线程安全问题
        • 解决方法：尽量不要在servlet中定义成员变量。即使定义了成员变量，也不要赋值
    • service方法
      • 执行service方法，执行多次
      • 每次访问servlet的时候，service都会被调用一次
    • destory方法
      • 只执行一次
      • 只有服务器正常关闭的时候，才会执行destory方法。
      • 在servlet被销毁之前执行。
    • 获取servlet.config对象
    • getServletInfo

    servlet3.0

    • 优点：不需要web.xml，使用注解配置
    • 步骤：
      • 创建JAVAEE项目，选择servlet版本3.0以上
      • 定义一个类，实现servlet接口
      • 重写方法
      • 在类上使用@webServlet注解，进行配置

Filter

过滤器细节

  • 通过web.xml或者注解配置
  • 执行流程
    • 执行过滤器
    • 执行放行后的资源
    • 执行过滤器放行代码下的代码
  • 生命周期
    • init() 服务器启动后，创建filter对象，调用init方法，只执行一次
    • doFilter，每次请求被拦截都会执行
    • destory 服务器关闭，filter对象被销毁，只执行一次
  • 配置详解
    • 拦截路径配置
      • 具体的资源路径

Listener

  • 事件监听机制
  • servletContextListener
    • contextDestory()
    • contextInitialized()
  • 实现监听器的步骤
    • 定义一个类，实现ServletContextListener接口
    • 复写方法
    • 配置

代理模式

  • 为了实现功能增强
  • 基本概念
    • 真实对象
    • 代理对象
    • 代理模式
  • 实现方式
    • 静态代理：一个类文件描述代理模式
    • 动态代 理：在内存中形成代理类

会话技术

会话技术

  • 一次会话中包含多次请求和响应
  • 功能：在一次会话的范围内共享数据
  • 方式：
    • 客户端使用Cookie
    • 服务器端Session

cookie

  • 客户端会话技术，将数据保存到客户端

  • 使用

    • 创建 cookie绑定数据

      new Cookie(String name .String value)

    • 发送cookie对象

      response.addCookie(Cookie cookie)

    • 获取cookie

      Cookies[] request.getCookies()

response

响应消息格式

数据格式

  • 响应行
    • 组成 :协议，响应状态码，状态码描述
    • 响应状态码：服务器告诉浏览器本次请求和响应的一个状态
      • 状态码分为5类，分别以1-5开头
      • 1xx : 服务器接收客户端消息但是没有接收完成，发送1xx码
      • 2xx：成功
      • 3xx: 重定向,302(重定向),304(访问缓存)
      • 4xx：客户端错误，404(请求路径没有 相应的资源)
      • 5xx:服务器端错误
  • 响应头
  • 响应空行
  • 响应体

response对象

  • 功能：设置响应消息

    • 设置响应行：包括格式个状态码
    • 设置响应头：setHeader(String name, String value)
    • 设置响应体：
      • 获取输出流
      • 使用输出流

  • 重定向的特点：

    • 地址栏发生变化

    • 重定向可以访问其他服务器的资源

    • 重定向是两次请求，不能使用request对象共享数据

  • 服务器输出数据到浏览器

    • 获取字符输出流：response.getWriter()
    • 输出数据
    • 乱码问题：使用response.setContentType()

  • 服务器输出字节数据到浏览器

    • 获取字节输出流

      response.getOutputStream()

    • 输出数据

      write("String".getBytes())

  • 验证码

ServletContext 对象

  • 概念：代表整个web应用和服务器来通信
  • 获取
    • 通过request获取： requset.getServletContext()
    • 通过HttpServlet获取：this.getServletContext()
  • 功能
    • 获取MIME类型
      • String getMimeType(filename)可以获取
    • 共享数据
      • setArrtibute(String name ,Object value)
      • getAttribute(String name )
      • removeAttribute(String name )
    • 获取文件的服务器路径
      • getRealPath(String path)
