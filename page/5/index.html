详细servlet解析

servlet

体系结构

  • GenericServlet:实现了其他的方法（包括空实现），只将service() 方法仍作为抽象方法，继承的时候需要实现该方法
  • HttpServlet: 继承自GenericServlet，可以方便的判断请求方式(即post和get方式)，对Http协议的封装，简化操作。

servlet相关配置

  • urlpartten:Servlet访问路径
    • 一个Servlet可以定义多个访问路径：@webServlet({“/d1”,”dd1”,”ddd1”})
    • 路径定义规则 ：
      • /xxx
      • /xxx/xxx/x :多层路径
      • *.do

HTTP

  • 定义了客户端和服务器端通信的时候，发送数据的格式
  • 特点：
    • 基于TCP/IP 的高级协议
    • 默认端口：80
    • 基于请求/相应模型的，也就是说一起请求对应一次响应
    • 是无状态的：每次请求之间是相互独立的
    • 1.1版本可以复用链接
  • 请求消息数据格式
    • 请求行
      • 请求方式 请求url 请求协议/版本
        • get方式：请求参数在请求行中，url后面；url长度有限制；不安全
        • post方式：请求参数在请求体中；url长度没有限制；安全
    • 请求头
      • 键值的方式
      • 常见的请求头
        • User-Agent:浏览器版本信息
        • Referer:告诉服务器，当前请求从哪来，可以防止盗链
        • Connection:
    • 请求空行
      • 空行
    • 请求体
      • 请求参数
  • 响应消息数据格式

Request原理

  • tomcat服务器会根据请求url中的资源路径，创建对应的ServletDemo1的对象
  • tomcat服务器会创建request和response对象， request对象中封装请求消息数据
  • tomcat将request和response两个对象 传递给service方法，并调用该方法

request对象继承体系结构

  • ServletRequest(接口)
  • HttpServletRequest（接口，继承自ServletRequest）
  • RequestrFacade（由tomcat实现接口HttpServletRequest）

Request的功能

  • 获取请求消息数据
    • 获取请求行数据
      • 获取请求方式 String getMethod()
      • 获取虚拟目录 Srting getContextPath()
      • 获取Servlet路径 String getSercletPath()
      • 获取get方式请求参数 String getQueryString()
      • 获取请求url StringBuffer getRequestURL()
      • 获取协议及版本 String getProcol()
      • 获取客户机的IP地址 String getRemoteAddr()
    • 获取请求头数据
      • String getHeader(String name) 根据请求头
    • 获取请求体数据
      • 只有post请求方式才有请求体
      • 步骤
        • 获取流对象 BufferedReader getReader(): 获取字符输入流，只能获取字符；
        • 从流对象中拿出数据
  • 其他功能
    • 获取请求参数的通用方式
      • String getParameter(String name): 根据参数名称获取参数值
      • String[] getParameterValues(String name): 根据参数名称获取参数值数组
      • Enumeration getParameterName() : 获取请求参数名称
      • Map<String, String[]> getParameterMap() : 获取所有参数的map集合
    • 请求转发:在服务器内部的资源跳转方式
      • 步骤
        • 通过request对象获取请求转发器对象： RequestDispatcher getRequestDispatcher(String path)
        • 使用RequestDispatcher对象进行转发： forward()
      • 特点
        • 浏览器地址不变化
        • 只能转发到当前服务器的内部资源
        • 转发是一次请求
    • 共享数据
      • 域对象：一个有作用范围的对象，可以在范围内共享数据
      • request域 ：代表一次请求的范围，一般用于请求转发的多个资源中共享数据
      • 方法：
        • void setAttribute(String name,.Object obj) :存储数据
        • Object getAttribute(String name): 通过键获取值
        • void removeAttribute(String name): 通过键移除值
    • 获取ServletContext
      • ServletContext getServletContext()

Servlet

Servlet

  • 概念：运行在服务器端的小程序，是一个接口，它定义了Java类被浏览器访问到的规则

  • 快速入门

    • 创建JavaEE项目
    • 定义一个类，实现Servlet接口
    • 实现抽象方法
    • 配置Servlet

  • 运行原理

    • 服务器接收到浏览器端的请求后，解析请求的URL路径，获取访问的servlet的资源路径
    • 查找web.xml文件，是否有对应的标签体内容
    • 如果有，查找全类名
    • tomcat会将字节码文件加载进内存，并创建对象
    • 调用器方法

    servlet中的生命周期

    • init方法
      • 被创建，执行init方法，只执行一次
        • 默认情况下，第一次被访问时被创建
        • 在web.xml中使用标签配置创建时机
          • 在第一次启动时创建，配置使用负数
          • 配置为0或者正数的时候，当服务器启动的时候被创建
      • init方法只执行一次，说明一个servlet在内存中只存在一个对象，servlet时单例的。
        • 所以当多个用户同时访问的时候，可能存在线程安全问题
        • 解决方法：尽量不要在servlet中定义成员变量。即使定义了成员变量，也不要赋值
    • service方法
      • 执行service方法，执行多次
      • 每次访问servlet的时候，service都会被调用一次
    • destory方法
      • 只执行一次
      • 只有服务器正常关闭的时候，才会执行destory方法。
      • 在servlet被销毁之前执行。
    • 获取servlet.config对象
    • getServletInfo

servlet3.0

  • 优点：不需要web.xml，使用注解配置
  • 步骤：
    • 创建JAVAEE项目，选择servlet版本3.0以上
    • 定义一个类，实现servlet接口
    • 重写方法
    • 在类上使用@webServlet注解，进行配置
