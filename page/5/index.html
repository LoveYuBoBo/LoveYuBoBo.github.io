基于JWT的单点登录系统设计

JWT（json web token）是一个含签名并携带用户相关信息的加密串，页面请求校验登录接口时，请求头中携带JWT串到后端服务，后端通过签名加密串匹配校验，保证信息未被篡改

  • JWT因为不需要在服务端记录用户信息，轻量，可以用作单点登录系统设计
  • 通过签名，验证合法性，通过头部和载荷验证数据的完整性

Redis高可用

Redis高可用方案

主从复制

  • 主从读写分离
  • 增量复制，全量复制
  • 主节点宕机后，从节点角色不会变化
  • 主节点重启后恢复主节点的角色

哨兵模式

主从复制中，主节点宕机后从节点不能担任主节点的角色，因此，哨兵模式就是解决这个问题的

  • 后台监控主机是否故障，如果故障自动将从节点升级为主节点

  • 如果哨兵出现故障，就无法进行监控了，可以为哨兵也建立集群

cluster

  • 一致性哈希
  • 主从结构保证高可用

Redis持久化方式

由于Redis在内存中操作数据，一旦发生断电或者宕机，数据会丢失，因此，Redis提供了将内存数据持久化到硬盘的方式

RDB

  • 是快照写入的方式

  • 自动触发和手动触发

  • 手动触发

    • save：使用改命令会阻塞当前redis服务，知道rdb完成
    • bgsave： fork子进程异步进行快照，此时写操作，将数据写到缓冲区中，快照结束后，再将缓冲区数据快照。

  • 恢复数据和停止rdb持久化

    • 将备份文件移动到redis安装目录并启动服务即可恢复数据
    • 注释掉redis.conf中所有save行即可停用

  • 优劣势

    • 优势

    　　RDB是一个非常紧凑(compact)的文件，它保存了redis 在某个时间点上的数据集。这种文件非常适合用于进行备份和灾难恢复。

    　 生成RDB文件的时候，redis主进程会fork()一个子进程来处理所有保存工作，主进程不需要进行任何磁盘IO操作。

    　 RDB 在恢复大数据集时的速度比 AOF 的恢复速度要快。

    • 劣势

    　　RDB方式数据没办法做到实时持久化/秒级持久化。因为bgsave每次运行都要执行fork操作创建子进程，属于重量级操作，如果不采用压缩算法(内存中的数据被克隆了一份，大致2倍的膨胀性需要考虑)，频繁执行成本过高(影响性能)

    　　RDB文件使用特定二进制格式保存，Redis版本演进过程中有多个格式的RDB版本，存在老版本Redis服务无法兼容新版RDB格式的问题(版本不兼容)

    　 在一定间隔时间做一次备份，所以如果redis意外down掉的话，就会丢失最后一次快照后的所有修改(数据有丢失)

Redis中数据结构

redis中常用的数据结构由string list hash set sorted set，还有比如sub/pub等

各种数据结构使用场景

string
  • 计数，由于Redis单线程的特点，我们不用考虑并发造成计数不准的问题，通过 incrby 命令，我们可以正确的得到我们想要的结果
  • 限制次数，比如登录次数校验，错误超过三次5分钟内就不让登录了，每次登录设置key自增一次，并设置该key的过期时间为5分钟后，每次登录检查一下该key的值来进行限制登录。
hash
  • 查询时间复杂度是O(1)，常用作缓存
list
  • 栈，通过命令 lpush+lpop
  • 队列，命令 lpush+rpop
  • 有限集合，命令 lpush+ltrim
  • 消息队列，命令 lpush+rpop
set
  • 利用set的特性，可以方便的获取多个用户的共同好友等
sorted set
  • 利用排序的特点，可以做排行榜等

搜索

空查询及参数

  • 使用不带参数_search，返回结果中包含hits,took,shards,timeout等信息
  • 空查询会查询出集群中所有的文档

多索引，多类型

  • 指定索引查询
  • 当在单一的索引下进行搜索的时候，Elasticsearch 转发请求到索引的每个分片中，可以是主分片也可以是副本分片，然后从每个分片中收集结果。多索引搜索恰好也是用相同的方式工作的，只是会涉及到更多的分片

分页

  • 使用from和size指定查询分页，个请求经常跨越多个分片，每个分片都产生自己的排序结果，这些结果需要进行集中排序以保证整体顺序是正确的
  • 分布式系统中深度分页，在分布式系统中，对结果排序的成本随分页的深度成指数上升。这就是 web 搜索引擎对任何查询都不要返回超过 1000 个结果的原因。

映射和分析

  • 不同类型的字段的索引方式不完全相同
  • 倒排索引由文档中所有不重复的词的列表构成，对于每个词，由一个包含它的文档列表
