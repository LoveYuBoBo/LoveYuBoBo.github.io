Redis中数据结构

redis中常用的数据结构由string list hash set sorted set，还有比如sub/pub等

各种数据结构使用场景

string
  • 计数，由于Redis单线程的特点，我们不用考虑并发造成计数不准的问题，通过 incrby 命令，我们可以正确的得到我们想要的结果
  • 限制次数，比如登录次数校验，错误超过三次5分钟内就不让登录了，每次登录设置key自增一次，并设置该key的过期时间为5分钟后，每次登录检查一下该key的值来进行限制登录。
hash
  • 查询时间复杂度是O(1)，常用作缓存
list
  • 栈，通过命令 lpush+lpop
  • 队列，命令 lpush+rpop
  • 有限集合，命令 lpush+ltrim
  • 消息队列，命令 lpush+rpop
set
  • 利用set的特性，可以方便的获取多个用户的共同好友等
sorted set
  • 利用排序的特点，可以做排行榜等

elasticSearch学习（四）

发表于 2020-07-09 | 分类于 elasticSearch
字数统计字: 613 | 阅读时长分钟 ≈ 2

搜索

空查询及参数

  • 使用不带参数_search，返回结果中包含hits,took,shards,timeout等信息
  • 空查询会查询出集群中所有的文档

多索引，多类型

  • 指定索引查询
  • 当在单一的索引下进行搜索的时候，Elasticsearch 转发请求到索引的每个分片中，可以是主分片也可以是副本分片，然后从每个分片中收集结果。多索引搜索恰好也是用相同的方式工作的，只是会涉及到更多的分片

分页

  • 使用from和size指定查询分页，个请求经常跨越多个分片，每个分片都产生自己的排序结果，这些结果需要进行集中排序以保证整体顺序是正确的
  • 分布式系统中深度分页，在分布式系统中，对结果排序的成本随分页的深度成指数上升。这就是 web 搜索引擎对任何查询都不要返回超过 1000 个结果的原因。

映射和分析

  • 不同类型的字段的索引方式不完全相同
  • 倒排索引由文档中所有不重复的词的列表构成，对于每个词，由一个包含它的文档列表

文档

文档元数据

  • _index：表明文档存放在哪里
  • _type: 文档表示的对象类别
  • _id文档唯一标识

索引文档

  • 通过提供id

  • 不添加id的时候，使用自增id

取回文档

  • 使用get,带上元数据参数，get请求会返回整个文档
  • 获取部分文档，可以通过_source指定返回的字段
  • 使用_mget批量获取文档

检查文档是否存在

  • 使用head代替get,只返回一个状态码

更新文档

  • 文档一旦创建是不能更改的，如果需要更新文档，只能重新索引，旧的文档也不会立刻删除，只是无法访问到

创建新文档

  • 通过元数据唯一表示一个文档，在创建新文档的时候，通过元数据来区别，判断是否已经存在相同的文档，也可以使用自增id

elasticSearch学习（二）

发表于 2020-07-08 | 分类于 elsticSearch
字数统计字: 1k | 阅读时长分钟 ≈ 3

mapping定义

注意事项

  • 只有使用post命令写入文档，es才会自动为其创建id，而使用put的时候不会创建
  • update一个文档，只能使用post，put只能用来index或者create
  • index一个已经存在的文档，旧的文档会被删除，新的文档再被写入且版本号加一
  • 当创建一个新的文档到不存在的索引中，默认情况会创建相应的索引，且可以设置mapping
  • ES7中合法的type是_doc
  • 精确值和全文本的本质区别在于是否会被Analyzer分词
  • Analyzer由三部分组成，分别是Character filter+Tokenizer+Token Filter
  • match和match_phrase的区别，``match中terms之间是or的关系，而match_phrase的terms是and的关系且term`之间的关系也会影响查询
  • 如果希望match_phrase匹配更多的结果，可以增加slot参数
  • 如果mapping的dynamic设置为stric，索引一个包含新新增字段的文档时会报错
  • 如果mapping的dynamic设置为false，索引一个包含新新增字段的文档时，文档会被索引，且新字段在_source中可见，但是该字段无法被搜索。
  • 不能所以修改字段类型，修稿字段类型都会造成reindex
  • 可以在mapping文件中为indexing和searching指定不同的Analyzer
  • 字段类型为text的时候，也不一定会被全文搜索，因为可以为其字段指定 not index
