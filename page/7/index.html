指令系统

机器指令

  • 指令的格式

    • 操作码 地址码 寻址方式

      • 操作码：反映机器做什么操作

        • 长度固定，用于指令字长较长的情况，如RISC

        • 长度可变，操作码分散再指令字的不同字段中

        • 扩展操作码，操作码的位数随地址数的减少而增加

          • 短操作码不能是长操作码的前缀

      • 地址码

        ​

    • 指令字长

      • 指令字长固定
      • 指令字长可变

    • 当用一些硬件资源代替指令字中的地址吗字段后

      • 可扩大指令的寻址范围
      • 可缩短指令字长
      • 可减少访存次数

    • 当指令的地址字段为寄存器时

操作数类型和操作类型

操作数类型
  • 地址，无符号数
  • 数字，定点数，浮点数，十进制数
  • 字符，ASCII
  • 逻辑数，逻辑运算
数据再存储器中的存放方式
  • 字节编址，数据在存储器中的存放方式
    • 在任意位置开始存储
      • 优点：不浪费存储资源
      • 缺点：除了访问一个字节之外，方位其他任何类型的数据都可能花费两个存储周期的时间，读写控制比较复杂
    • 从一个存储字的起始地址开始
      • 优点：无论访问何种类型的数据，在一个周期均可完成，读写控制简单
      • 缺点：浪费了存储资源
    • 边界对准方式：从地址的整数倍位置开始访问
操作类型

  • 数据传送

  • 算术逻辑操作

  • 移位操作

  • 转移

    • 无条件转移 JMP
    • 条件转移
    • 调用和返回
    • 陷阱与陷阱指令
      • 一般不提供给用户直接使用

  • 输入输出

寻址方式

数据寻址

系统总线

总线的基本概念

  • 为什么使用总线
  • 总线
    • 总线是连接各个部件的信息传输线，是各个部件共享的传输介质
  • 总线上信息的传送
    • 串行
    • 并行
  • 总线结构的计算机例子
    • 单总线
    • 面向CPU的双总线，提供CUP到主存的专用总线，主存不和系统总线相连
  • 以存储器为中心的双总线
    • 主存和系统总线相连

总线的分类

  • 片内总线：芯片内部的总线
  • 系统总线：计算机各部件之间的信息传输线
    • 数据总线：双向，与机器字长、存储字长有关
    • 地址总线：单向，与存储地址，I/O地址有关
    • 控制总线
  • 通信总线：用域计算机之间或计算机系统与其他系统之间的通信
    • 并行通信
    • 串行通信

总线的特性及性能指标

  • 总线物理实现
  • 总线特性
    • 机械特性
      • 尺寸，形状，管脚数等
    • 电气特性
      • 传输方向和有效电平的范围
    • 功能特性
      • 每根传输线的功能
    • 时间特性
      • 信号的时序关系
  • 总线的性能指标
    • 总线宽度：数据线的根数
    • 标准传输率：每秒传输的最大字节数
    • 时钟同步/异步
    • 总线复用：地址线与数据线复用
    • 信号线数：地址数，数据线和控制线的总和
    • 总线控制方式
    • 其他：负载能力
  • 总线标准

总线结构

  • 双总线结构（通过通道连接，由通道对I/O统一管理）
    • 主存总线
    • I/O总线
  • 三总线结构
    • I/O总线
    • 主存总线
    • DMA总线
  • 另一种三总线结构
    • 局部总线
    • 系统总线
    • 扩展总线
  • 四总线结构
    • 局部总线
    • 系统总线
    • 高速总线
    • 扩展总线（连接到高速总线）

总线控制

  • 总线判优控制

    • 基本概念
      • 主设备：对总线由控制权
      • 从设备：响应从主设备发来的总线命令
      • 总线判优控制：
        • 集中式
          • 链式查询：对链路故障很敏感
            • BS：总线忙
            • BR：总线请求
            • BG：总线同意
          • 定时器定时查询（优先级判断很灵活（从计数器计数开始））
            • 设备地址线
            • BS
            • BR
          • 独立请求方式
            • 每一个I/O接口都增加了BG和BR
            • 通过总线控制部件控制优先级
        • 分布式

  • 总线通信控制

    • 目的：解决通信双方协调配合问题
    • 总线传输周期
      • 申请分配阶段：主模块申请，总线仲裁决定
      • 寻址阶段：主模块向从模块给出地址和命令
      • 传数阶段：主模块和从模块交换数据
      • 结束阶段：主模块撤销有关信息
    • 总线通信的方式
      • 同步通信：由统一（定宽）时标控制数据传送
      • 异步通信：采用应答方式，没有公共时钟标准
      • 半同步通信：同步异步结合
      • 分离式通信：充分挖掘系统总线每个瞬间的潜力
    • 一个总线传输周期
      • (同步，异步，半异步)
        • 主模块发送地址、命令：占用总线
        • 从模块准备数据：不占用总线
        • 从模块向主模块发数据：占用总线
      • 分离式通信
        • 子周期1：主模块申请占用总线，使用完后即放弃总线的使用权
        • 子周期2：从模块申请占用总线，将各种信息送至总线上
    • 分离式通信的特点
      • 各模块有权申请占用总线
      • 采用同步方式通信，不等对方回答
      • 各模块准备数据时，不占用总线
      • 总线被占用时，无空闲
