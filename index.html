Mybatis框架2

Mybatis框架

连接池

  • 连接池就是用于存储链接的容器，可以减少获取链接消耗的时间
  • 连接池配置方式
    • 在主配置文件SqlMapConfig.xml中dataSource标签，type属性表示采用何种连接池方式
    • type属性的取值
      • POOLED：采用传统的javax.sql.DataSource规范中的连接池，每次从池中获取链接使用
      • UNPOOLED：采用传统获取链接的方式，但是没有实现连接池。每次创建一个新的链接
      • JNDI:采用服务器提供的JNDI技术实现，获取DataSource对象，如果不是web或者maven工程，不能使用

事务

  • 什么是事务
  • 事务的四大特性（原子性，一致性，隔离性，持久性）
  • 不考虑隔离性会产生的三大问题（脏读，不可重复读，幻读）
  • 隔离级别（serializable,repeated read(Mysql默认级别),read commited,read uncommited）

自动提交设置

  • 在openSession()中设置为true

动态sql

单一条件查询
  • 配置中使用if标签，拼接SQL语句
  • 使用where标签，嵌套if标签
多条件查询
  • 使用foreach标签，有collection open close item seperate等属性
抽取重复的标签
  • 使用sql标签

Mybatis框架

Mybatis框架

持久层技术解决方案

  • JDBC技术
    • Connection
    • PreparedStatement
    • ResultSet
  • Spring中的JdbcTemplate
  • Apache的DBUtils

mabatis概述

  • 使用了ORM的思想实现对结果集的封装
  • 环境搭建
    • 创建maven工程并导入坐标
    • 创建实体类和dao接口
    • 创建Mybatis主配置文件
    • 创建映射配置文件
  • 注意
    • mabatis的映射配置文件位置必须和dao接口的包结构相同
    • 映射配置文件中的namespace属性的取值必须是dao接口的全限定类名
    • 映射文件的配置。id属性的取值必须是dao接口的方法名

配置步骤

  • 配置Mysql环境
    • 配置事务 transcationManager标签
    • 配置连接池dataSource
    • 配置映射文件位置 mappersb标签的resource属性

使用步骤

  • 读取配置文件
  • 创建SqlSessionFactory
  • 使用工厂生产SqlSession对象: factory.oepnSession
  • 使用SqlSession创建dao接口的代理对象：session.getMapper()
  • 使用代理对象执行方法
  • 释放资源

注意事项

  • 读取配置文件的时候(不使用相对路径和绝对路径)
    • 使用类加载器，只能加载类路径的配置文件
    • 使用ServletContext对象的getRealPath()
  • Mybatis创建工厂使用了构建者模式，使用SqlSessionFactoryBuilder创建工厂，使用者直接调用方法即可拿到对象
  • 在生产SqlSession的时候使用了工厂模式，降低了类之间的依赖
  • 创建dao接口实现类使用了代理模式，在不修改源码的基础上实现功能增强

自定义Mybatis

  • 执行的流程
    • 根据配置文件的信息创建connection对象，注册驱动，获取连接
    • 获取预处理对象preparedStatement，此处需要SQL语句
    • 执行查询
    • 遍历结果用于封装，此处用配置文件中的全限定类名，使用反射创建对象并封装
  • 自定义Mybatis涉及的类
    • class Resources
    • class SqlSessionFactoryBuilder
    • interface SqlSessionFactory

CRUD操作

Dao实现类的执行过程

  • PreparedStatement对象的方法
    • excute只能执行CRUD中任意一种,返回值是布尔值，表示是否有结果集
    • excuteUpdate只能执行CUD语句。返回值是影响数据库的行数
    • excuteQuery只能执行Select语句，返回结果集 resultSet

Dao代理的执行过程

解决实体类和数据库列名不对应的情况
  • sql语句中使用别名
  • 使用resultMap标签
  • typeAlias指定别名，属性是domain中的实体类
insert
  • 使用insert标签注册事件并制定实体类的全限定类名，加sql语句
  • 使用SqlCommit保存数据操作
  • 获取插曲用户的id
    • 使用keyPropety属性，并且使用sql语句select last_insert_id()
update
  • 使用update标签配置并制制定类型，加上sql语句
delete
  • 使用delete标签配置
select

  • 使用select标签并指定返回的数据类型

  • 模糊查询传入%，保证sql语句的正确性

SpringMVC

SpringMVC

三层架构

和Structs2的异同点

  • 相同点
    • 底层都是基于原始Servlet的
    • 都是基于MVC的
    • 处理请求的机制都是一个核心控制器
  • 不同点
    • 入口不同：SpringMVC入口是Servlet，Structs2入口是Filter
    • SpringMvVC基于方法，Structs2基于类

常用注解

  • RequestMapping
    • value和path属性均可指定请求路径
    • method方法指定请求方式
    • params 指定请求参数的条件
    • headers指定限制请求消息头的条件
  • RequestParam 请求中指定名称的参数给控制器中的形参赋值
  • RequestBody：用于获取请求体的内容，get方法不适用
  • PathVaribale 用于绑定url中的占位符
  • RequestHeader 获取请求头消息
  • CookieValue 用于获取指定cookie的值
  • ModelArrtibute
  • SessionAttribute 作用在类上

请求参数的绑定

  • controller使用和前端传递的参数一致的参数接收
  • form表单提交的时候name属性的值应该和JavaBean中属性值一致（使用JavaBean中的set方法封装）
  • 引用类型的封装：使用表达式
  • 解决中文乱码的问题，配置过滤器

类型转换

  • 类型转换需要实现Converter接口
  • 配置自定义类型转换器
    • ConversionServiceFactory

获取原生的servlet API

  • 方法上加参数

响应

  • 响应是字符串

    • 手动调用model，将参数封装

  • 返回值是void

    • 使用转发，注意：手动转发的时候不会使用视图解析器
    • 可是使用重定向的时候
    • 直接进行相应，使用response.getWrite();

  • 返回的是ModelAndView

  • 返回的是json数据

动态代理

动态代理

基于接口的动态代理

  • 要求被代理类至少实现一个接口

  • 使用JDK提供的Proxy类

  • 使用Proxy的newProxyInstance方法创建代理对象

  • 创建代理对象的方法有三个参数，CLassLoader用于指定代理对象的字节码，CLass[] 用于让代理对象和被代理对象有相同的方法，InvocationHandler用于提供增强的代码，一般是该接口的实现类

基于子类的动态代理

  • 要求被代理类不能是最终类
  • 由第三方cglib提供的Enhance类
  • 使用Enhance的create方法
  • create方法有两个参数，Class指定被代理对象的字节码，callback用于提供增强的代码。
